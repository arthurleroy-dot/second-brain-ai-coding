# Top engineers at Anthropic, OpenAI say AI now writes 100% of their code

## Résumé
Article de *Fortune* (janvier 2026) rapportant que des ingénieurs de premier plan dans les grands labs d'IA n'écrivent plus leur code à la main. Boris Cherny, chef de Claude Code chez Anthropic, déclare que 100 % de son code est désormais écrit par Claude Code et Opus 4.5 ; « Roon » (chercheur OpenAI) affirme la même chose. L'article met ces affirmations en perspective avec les chiffres plus modestes du reste de l'industrie (Microsoft, Salesforce, étude *Science*), la position officielle d'Anthropic (70-90 % à l'échelle de l'entreprise), et les implications sur l'emploi — en particulier les postes juniors et l'évolution du métier d'ingénieur vers l'édition/orchestration.

## Contenu complet

Chez Anthropic — un lab d'IA construisant certains des modèles les plus avancés du monde — les ingénieurs n'écrivent plus le code qui alimente leurs produits ; ils le délèguent à l'IA. Le chef de Claude Code chez Anthropic, Boris Cherny, a annoncé qu'il n'a écrit aucun code depuis plus de deux mois.

Dans un post sur X, Cherny a dit que 100 % de son code est désormais écrit par Claude Code et Opus 4.5 d'Anthropic. Dans le reste de l'entreprise, il affirme que « pretty much 100% » (à peu près 100 %) du code est aussi généré par l'IA.

« For me personally, it has been 100% for two+ months now, I don't even make small edits by hand » (« Pour moi personnellement, c'est 100 % depuis plus de deux mois maintenant, je ne fais même pas de petites modifications à la main »), a écrit Cherny dans un post sur X en réponse au chercheur en IA Andrej Karpathy. « I shipped 22 PRs (pull requests) yesterday and 27 the day before, each one 100% written by Claude » (« J'ai expédié 22 PR hier et 27 l'avant-veille, chacune écrite à 100 % par Claude »).

Ces commentaires font écho aux propos antérieurs du CEO d'Anthropic, Dario Amodei, au Forum économique mondial plus tôt dans le mois, où il a noté que certains ingénieurs de son entreprise ont cessé d'écrire eux-mêmes du code et s'appuient plutôt sur des modèles d'IA pour le générer, pendant qu'ils se concentrent sur l'édition. À Davos, Amodei a prédit que l'industrie pourrait n'être qu'à six à douze mois d'une IA gérant l'essentiel ou la totalité du travail de génie logiciel, de bout en bout.

Cherny n'est pas le seul ingénieur en vue à annoncer qu'il a largement abandonné le codage manuel. Roon, un compte pseudonyme populaire sur X tenu par un chercheur d'OpenAI, a aussi déclaré dans un post sur X qu'il n'écrit plus aucun de ses propres codes. « 100%, I don't write code anymore » (« 100 %, je n'écris plus de code »), a écrit l'utilisateur lorsqu'on lui a demandé quel pourcentage de son codage est fait par des modèles d'IA. Dans un post séparé, il a ajouté : « Programming always sucked. It was a requisite pain for ~everyone who wanted to manipulate computers into doing useful things, and I'm glad it's over » (« Programmer a toujours été nul. C'était une peine obligatoire pour ~tous ceux qui voulaient amener les ordinateurs à faire des choses utiles, et je suis content que ce soit fini »).

Bien que les acteurs de l'industrie aient des motivations à survendre leurs propres outils, il existe un consensus grandissant selon lequel l'industrie a déjà été fondamentalement transformée par l'essor des outils de codage par IA.

« The whole way people build software has changed; software is not like what it used to be » (« Toute la manière dont les gens construisent des logiciels a changé ; le logiciel n'est plus ce qu'il était »), a déclaré Himanshu Tyagi, cofondateur de la société d'AGI open-source Sentient, à *Fortune*. « A large part of the code that will be shipped over the next 10 years will be written by AI… [Anthropic's] Claude Code is the breakthrough product that has made that happen » (« Une grande partie du code qui sera expédié au cours des 10 prochaines années sera écrite par l'IA… Claude Code [d'Anthropic] est le produit de rupture qui a rendu cela possible »).

Pourtant, en dehors des principaux labs d'IA, les chiffres de code généré par IA rapportés par beaucoup d'entreprises de logiciels sont significativement plus bas. Par exemple, le CEO de Microsoft, Satya Nadella, a déclaré en avril 2025 que l'IA générait environ 30 % du code chez le géant du logiciel. Salesforce a avancé un chiffre similaire. Une étude publiée dans la revue *Science* plus tôt dans le mois a examiné des fonctions Python sur GitHub et a constaté qu'environ 29 % aux États-Unis sont désormais écrites par l'IA, avec des pourcentages plus faibles dans d'autres régions.

Et bien que Cherny dise que l'IA écrit désormais 100 % de son code, un porte-parole d'Anthropic a indiqué qu'à l'échelle de l'entreprise le chiffre se situe entre 70 % et 90 %. Pour Claude Code, environ 90 % de son code est écrit par Claude Code lui-même.

Cherny, lui, croit que ces chiffres vont continuer à grimper, et que d'autres entreprises commenceront bientôt à atteindre des niveaux similaires de génération de code par IA. « I think most of the industry will see similar stats in the coming months—it will take more time for some vs others » (« Je pense que la plupart de l'industrie verra des statistiques similaires dans les mois à venir — ce sera plus long pour certains que pour d'autres »), a-t-il écrit. « We will then start seeing similar stats for non-coding computer work also » (« Nous commencerons alors à voir des statistiques similaires pour le travail informatique non lié au codage »).

Les outils d'Anthropic sont devenus un favori des ingénieurs logiciels ces dernières années. Mais la sortie de Claude Code a résonné auprès des codeurs comme des non-codeurs et a déclenché un moment viral pour l'entreprise, comme on n'en avait pas vu depuis les débuts de ChatGPT. Après que des utilisateurs ont fait remarquer que Claude Code était davantage un agent d'IA généraliste, Anthropic a créé une version du produit pour les non-codeurs, lançant Cowork, un agent de gestion de fichiers qui est essentiellement une version conviviale du produit de codage. Cherny a dit que son équipe a construit Cowork en environ une semaine et demie, en grande partie en utilisant Claude Code lui-même.

Même avant la frénésie publique, Cherny dit que l'outil faisait déjà des vagues au sein de leur propre entreprise.

« Somewhere around a year ago…we had this idea that the model was powerful enough that we could use it for a different kind of coding…we started to try out internally, and it just immediately took off » (« Il y a environ un an… nous avons eu cette idée que le modèle était assez puissant pour qu'on puisse l'utiliser pour un autre type de codage… nous avons commencé à l'essayer en interne, et ça a immédiatement décollé »), a dit Cherny à *Fortune* lors d'un entretien la semaine dernière. « I have never had this much joy day to day in my work, as I do right now, because essentially all the tedious work, Claude does it, and I get to be creative. I get to think about what I want to build next » (« Je n'ai jamais eu autant de plaisir au quotidien dans mon travail que maintenant, parce qu'essentiellement tout le travail fastidieux, Claude le fait, et je peux être créatif. Je peux réfléchir à ce que je veux construire ensuite »).

Cherny a dit qu'il utilise aussi Claude Code pour divers aspects administratifs en dehors du codage, y compris des tâches de gestion de projet comme envoyer automatiquement des messages aux membres de l'équipe sur Slack lorsqu'ils n'ont pas mis à jour des feuilles de calcul partagées.

« Engineers just feel unshackled, that they don't have to work on all the tedious stuff anymore » (« Les ingénieurs se sentent simplement libérés, ils n'ont plus à travailler sur tout ce qui est fastidieux »), a-t-il dit.

### A reckoning for the software industry (Un moment de vérité pour l'industrie du logiciel)

L'essor du code généré par l'IA a significativement impacté l'industrie du logiciel. Beaucoup d'entreprises Big Tech ont été ouvertes sur le fait que des modèles d'IA écrivent des quantités importantes de leur code. Mais l'automatisation d'une grande partie du processus de codage a aussi soulevé des questions sur l'avenir des rôles de génie logiciel, en particulier les postes de niveau débutant qui ont traditionnellement servi de terrain de formation pour la profession.

Les entreprises tech soutiennent que l'adoption rapide d'outils de codage par IA comme Claude Code et GitHub Copilot va démocratiser le codage, permettant à ceux qui ont peu ou pas de compétences techniques de construire des produits en sollicitant des systèmes d'IA en langage naturel. Mais, bien qu'il ne soit pas établi que les deux soient liés de manière causale — et qu'il existe d'autres facteurs impactant le déclin d'un emploi —, les postes ouverts pour les ingénieurs logiciels juniors ont effectivement décliné à mesure que la quantité de code écrit par l'IA générative s'est intensifiée.

Ce changement modifie déjà la façon dont Anthropic aborde le recrutement. Cherny a dit que son équipe recrute désormais surtout des généralistes plutôt que des spécialistes, puisque beaucoup de compétences de programmation traditionnelles sont moins pertinentes quand l'IA gère les détails d'implémentation.

« Not all of the things people learned in the past translate to coding with LLMs » (« Tout ce que les gens ont appris dans le passé ne se transpose pas au codage avec des LLM »), a écrit Cherny. « The model can fill in the details » (« Le modèle peut remplir les détails »).

Bien que Cherny ait insisté sur les gains de productivité et la liberté créative qu'apportent les outils de codage par IA, il a aussi reconnu que la technologie est encore en développement. Selon l'évaluation de Karpathy, les modèles peuvent commettre des « subtle conceptual errors » (« erreurs conceptuelles subtiles »), sur-compliquer le code et laisser du code mort traîner. Malgré les limites, des ingénieurs comme Cherny sont convaincus que la qualité du code généré par l'IA ne fera que continuer à s'améliorer.

## Concepts clés
- « 100 % du code écrit par l'IA » dans les labs (Cherny chez Anthropic, Roon chez OpenAI)
- Opus 4.5 et Claude Code comme outils de production internes
- Délégation du codage à l'IA, l'ingénieur se recentrant sur l'édition et la créativité
- Prédiction d'Amodei (Davos / Forum économique mondial) : 6-12 mois avant une automatisation large du génie logiciel de bout en bout
- Écart entre les labs de pointe et le reste de l'industrie (Microsoft ~30 %, Salesforce similaire, étude *Science* ~29 % aux É.-U.)
- Position officielle d'Anthropic : 70-90 % du code à l'échelle de l'entreprise, ~90 % pour Claude Code lui-même
- Extension annoncée vers le « travail informatique non lié au codage »
- Cowork : version pour non-codeurs de Claude Code (agent de gestion de fichiers), construit en ~1,5 semaine avec Claude Code
- Claude Code comme « produit de rupture » (breakthrough product) et moment viral comparable au lancement de ChatGPT
- Démocratisation du codage par le langage naturel
- Impact sur l'emploi : déclin des postes juniors, terrain de formation traditionnel menacé
- Évolution du recrutement : généralistes plutôt que spécialistes ; compétences traditionnelles moins pertinentes
- Limites reconnues (Karpathy) : erreurs conceptuelles subtiles, sur-complexification, code mort
- Usage de Claude Code pour des tâches admin/gestion de projet (ex. relances Slack)
